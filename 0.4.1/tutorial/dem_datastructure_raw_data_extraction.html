DEM Data Structure: Raw Data Extraction

A practical guide to extracting raw numerical data from DEM models for external processing, analysis, or visualization. Learn how to convert geometry, centroids, contacts, and connectivity information into NumPy arrays for computational workflows.

Contents

  1. Model Setup: create a compas_dem.models.BlockModel from compas_dem.templates.ArchTemplate and compute contacts.

  2. Mesh Data: extract vertices and faces as NumPy arrays from element geometries.

  3. Centroids: convert element centroids to a structured NumPy array.

  4. Contact Polygons: extract contact geometry as a list of NumPy arrays.

  5. Contact Indices: get connectivity information as element index pairs.

Imports

from typing import List
from typing import Tuple

import numpy as np

from compas_dem.models import BlockModel
from compas_dem.templates import ArchTemplate

Model Setup

We start by creating a model from an compas_dem.templates.ArchTemplate and computing contacts using compas_dem.models.BlockModel.compute_contacts(). This provides the foundation data structure for raw data extraction.

template: ArchTemplate = ArchTemplate(rise=3, span=10, thickness=0.5, depth=0.5, n=50)
model: BlockModel = BlockModel.from_template(template)
elements: List = list(model.elements())
model.compute_contacts(tolerance=0.001)

Mesh Data

Extract vertices and faces from each element’s geometry and convert them to NumPy arrays. Vertices are stored as float64 arrays containing 3D coordinates, while faces are stored as int32 arrays containing vertex indices.

numpy_vertices: List[np.ndarray] = []
numpy_faces: List[np.ndarray] = []
for element in elements:
    V, F = element.modelgeometry.to_vertices_and_faces()
    numpy_vertices.append(np.asarray(V, dtype=np.float64))
    numpy_faces.append(np.asarray(F, dtype=np.int32))

Note

The resulting numpy_vertices and numpy_faces are lists of variable-sized arrays, as each element can have a different number of vertices and faces.

Centroids

Element centroids are extracted and converted to a single NumPy array for efficient processing. Each row represents the 3D centroid coordinates (3 floats per row) of one element.

centroids: List = []
for element in elements:
    centroids.append(element.modelgeometry.centroid())
numpy_centroids: np.ndarray = np.asarray(centroids, dtype=np.float64)

Contact Polygons

Contact polygons represent the geometric interfaces between elements. Each contact is converted to a NumPy array containing the polygon vertices.

numpy_contact_polygons: List[np.ndarray] = []
for contact in model.contacts():
    numpy_contact_polygons.append(np.asarray(contact.polygon, dtype=np.float64))

Note

The numpy_contact_polygons list contains variable-sized arrays, as contact polygons can have different numbers of vertices.

Contact Indices

Extract connectivity information (2 integers per row) as pairs of element indices that are in contact. This provides the graph structure of the assembly.

contact_indices: List[Tuple[int, int]] = []
for edge in model.graph.edges():
    contacts = model.graph.edge_attribute(edge, name="contacts")
    if contacts:
        contact_indices.append(edge)
numpy_contact_indices: np.ndarray = np.asarray(contact_indices, dtype=np.int32)
